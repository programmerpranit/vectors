TESTING REDEFING

TESTING REDEFING is programming contest that helps the coders to build critical thinking , error solving .

Rules

1) Round 1 Code debugging:
Participant will get 3 problem which they have to solve within 20 minutes.
Rules:- Only logical error and variable name change is allowed. Participant should not make any syntax error or erase any line in code.Any additional rules will be told during event.
Round 2 Code debugging and Optimizing :
Participant is required to solve 2 out of 3 problem statement given in 40 minutes .
Code can be of this language :C,Java ,Python.
Rules:- Only logical error and variable name change is allowed. Participant should not make any syntax error or erase any line in code.Any additional rules will be told during event.
Judging criteria:-

Based on test cases decided by organizers.
Total time by participants to solve. Note :- Decision by the organizers will be final.
ROUND 3 DSA (Code debugging and Optimizing):
Participant will get 2 question out of 2 they need to solve 1 within 1 hour.
This round questions will based on DSA. Rules:- Only logical error and variable name change is allowed. Participant should not make any syntax error or erase any line in code.Any additional rules will be told during event.
Judging criteria:-
Based on test cases decided by organizers.
Total time by participants to solve.
